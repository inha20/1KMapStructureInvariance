X.X.1 좌표계 및 표기법 

본 연구에서는 4변수 카르노맵의 변수 배치 구조를 분석하기 위하여 좌표계와 표기법을 다음과 같이 정의한다.

카르노맵의 각 셀 위치는 (행, 열) 형태로 표기한다. 행 번호와 열 번호는 모두 좌측 상단을 기준으로 하며, 행 번호는 위에서 아래 방향으로 증가하고 열 번호는 왼쪽에서 오른쪽 방향으로 증가한다. 따라서 행과 열의 인덱스는 각각 1, 2, 3, 4의 순서로 부여된다.

[좌표계 정의 시각자료]

한편 카르노맵의 변수 배열은 일반적인 이진수 순서가 아닌 그레이코드(Gray Code) 순서를 사용한다. 2변수 그레이코드의 배열 순서는 다음과 같다.

00, 01, 11, 10

본 연구에서는 계산 및 대칭성 분석의 편의를 위하여 위의 그레이코드 순서에 대응하는 인덱스를 다음과 같이 정의한다.

00 → 0

01 → 1

11 → 3

10 → 2

따라서 카르노맵의 행 또는 열에 사용되는 그레이코드 인덱스 순서는

0, 1, 3, 2

로 표현된다.

[그레이코드 및 인덱스 대응표]

이후 변수 순서 교환을 분석하는 과정에서 그레이코드의 비트 순서를 반대로 읽는 경우가 발생한다. 예를 들어 변수 순서가 XY에서 YX로, 또는 ZW에서 WZ로 변경될 경우 기존의 그레이코드 순서

0, 1, 3, 2

는

0, 2, 3, 1

로 변환된다.

본 논문에서는 이러한 변환 과정을 명확히 설명하기 위하여 각 절에서 변수명, 그레이코드 값, 인덱스 사이의 대응 관계를 표로 정리하여 제시한다.

또한 카르노맵의 좌우 경계와 상하 경계는 서로 연결되어 있다고 가정한다. 따라서 행 또는 열 전체를 일정한 간격만큼 이동시키는 순환 이동(cyclic shift)은 카르노맵의 인접 관계를 유지하는 구조 보존 변환으로 취급한다. 이 가정은 이후 변수 순서 교환과 대칭성 분석의 기본 전제로 사용된다.

[순환 이동 개념 시각자료]


X.X.2 변수 순서 교환과 거울 대칭성

본 절에서는 변수 집합은 유지한 채 변수의 순서만 변경하는 경우를 분석한다. 변수 순서의 변화는 카르노맵의 형태를 변화시키는 것처럼 보이지만, 실제로는 거울 대칭과 순환 이동을 통하여 동일한 구조로 해석될 수 있음을 보인다.

정리 1. 행 변수 순서 교환

변수 배치가 XY/ZW일 때 행 변수의 순서를 교환한 YX/ZW는 원래 카르노맵과 동치이다.

[
XY/ZW \equiv YX/ZW
]

증명에 앞서 2변수 그레이코드의 순서를 살펴보자.

변수값| 인덱스
00| 0
01| 1
11| 3
10| 2

따라서 원래의 그레이코드 인덱스 순서는

[
0,1,3,2
]

로 표현된다.

행 변수의 순서를 XY에서 YX로 변경하면 각 비트의 위치가 교환되므로 인덱스 순서는

[
0,1,3,2
\rightarrow
0,2,3,1
]

로 변환된다.

[XY → YX 인덱스 변환표]

[XY/ZW 와 YX/ZW 비교 시각자료]

위 결과는 원래 배열과 다른 형태를 가지지만, 카르노맵의 상하 반사와 동일한 구조를 형성하며 순환 이동을 고려하면 동일한 인접 관계를 유지한다.

[상하 반사 시각자료]

[상하 반사 후 순환 이동 시각자료]

따라서 행 변수 순서 교환은 카르노맵의 구조를 변화시키지 않으며,

[
XY/ZW \equiv YX/ZW
]

가 성립한다.

정리 2. 열 변수 순서 교환

변수 배치가 XY/ZW일 때 열 변수의 순서를 교환한 XY/WZ는 원래 카르노맵과 동치이다.

[
XY/ZW \equiv XY/WZ
]

열 변수의 순서를 ZW에서 WZ로 변경하면 각 비트의 위치가 교환되므로 그레이코드 인덱스 순서는

[
0,1,3,2
\rightarrow
0,2,3,1
]

로 변환된다.

[ZW → WZ 인덱스 변환표]

여기에 카르노맵의 순환 이동을 적용하면

[
0,2,3,1
\rightarrow
2,3,1,0
]

을 얻는다.

한편 원래의 그레이코드 인덱스

[
0,1,3,2
]

를 역순으로 배열하면

[
2,3,1,0
]

이 된다.

즉 변수 순서 교환 후의 배열은 순환 이동을 고려할 경우 원래 배열의 좌우 반사와 동일한 결과를 가진다.

[
0,1,3,2
\rightarrow
0,2,3,1
\rightarrow
2,3,1,0
]

[0132 → 0231 → 2310 변환 시각자료]

[XY/ZW 와 XY/WZ 비교 시각자료]

[좌우 반사 대응 시각자료]

따라서 열 변수 순서 교환은 카르노맵의 구조를 변화시키지 않으며,

[
XY/ZW \equiv XY/WZ
]

가 성립한다.

이상의 결과로부터 행 변수 순서 교환과 열 변수 순서 교환은 모두 카르노맵의 구조를 보존하는 동치 변환임을 알 수 있다.


X.X.3 축 교환과 대각선 대칭성 

앞 절에서는 행 변수 또는 열 변수 내부의 순서만 변경하는 경우를 분석하였다. 본 절에서는 행 변수와 열 변수의 역할 자체를 서로 교환하는 경우를 다룬다.

변수 배치가

[ XY/ZW ]

일 때, 행 변수와 열 변수를 서로 교환하면

[ ZW/XY ]

를 얻는다.

이는 단순한 변수 순서 교환과는 다른 종류의 변환이며, 카르노맵 전체의 행축과 열축이 서로 교환되는 효과를 가진다.

정리 3. 축 교환 

변수 배치가 XY/ZW일 때 행 변수와 열 변수를 서로 교환한 ZW/XY는 원래 카르노맵과 동치이다.

[ XY/ZW \equiv ZW/XY ]

카르노맵의 각 셀 위치를

[ (a\text{행}, b\text{열}) ]

로 정의하자.

행 변수와 열 변수를 교환하면 기존의 행 인덱스는 새로운 열 인덱스가 되고, 기존의 열 인덱스는 새로운 행 인덱스가 된다.

따라서 셀의 위치는

[ (a,b) \rightarrow (b,a) ]

로 변환된다.

[행·열 교환 좌표 변환 시각자료]

이 변환은 카르노맵 배열에서 행과 열의 역할을 서로 교환하는 전치(transpose)에 해당한다.

이를 확인하기 위하여 4×4 배열의 각 셀에 서로 다른 문자를 부여한 예를 생각하자.

[16개 알파벳 배열 시각자료]

축 교환을 적용하면 각 셀은 행 번호와 열 번호를 서로 교환한 위치로 이동하게 되며, 결과적으로 대각선 방향의 대칭 구조가 형성된다.

[알파벳 배열 대각선 대칭 시각자료]

또한 동일한 과정을 그레이코드 배열에 적용하면 변수의 배치만 변화할 뿐 카르노맵의 인접 관계는 그대로 유지됨을 확인할 수 있다.

[그레이코드 배열 대각선 대칭 시각자료]

따라서 축 교환은 카르노맵의 본질적인 구조를 변화시키지 않는 동치 변환이며,

[ XY/ZW \equiv ZW/XY ]

가 성립한다.

한편 축 교환은 앞 절에서 다룬 변수 순서 교환과는 구별되는 독립적인 변환이다. 변수 순서 교환은 동일한 축 내부에서 변수의 순서만 변경하지만, 축 교환은 행 변수 집합과 열 변수 집합 자체를 서로 교환한다는 점에서 구조적으로 다른 의미를 가진다.

따라서 본 연구에서는 축 교환을 변수 순서 교환과 구별되는 독립적인 동치 변환으로 취급한다.

X.X.4 동치 변환의 정의 

앞 절에서 다음 세 가지 변환이 모두 카르노맵의 구조를 보존함을 확인하였다.

첫째, 행 변수 순서 교환

[ XY/ZW \equiv YX/ZW ]

둘째, 열 변수 순서 교환

[ XY/ZW \equiv XY/WZ ]

셋째, 행 변수와 열 변수의 교환

[ XY/ZW \equiv ZW/XY ]

이다.

본 연구에서는 위 세 가지 변환과 전역 순환 이동을 카르노맵의 동치 변환으로 정의한다.

즉 두 변수 배치가 위 변환들의 유한한 조합을 통하여 서로 변환될 수 있다면, 두 배치는 동일한 동치류(equivalence class)에 속한다고 정의한다.

[동치 변환 생성도]

예를 들어 변수 배치

[ XY/ZW ]

에 대하여 행 변수 순서 교환, 열 변수 순서 교환, 축 교환을 적용하면 다음과 같은 여러 형태를 얻을 수 있다.

[ XY/ZW ]

[ YX/ZW ]

[ XY/WZ ]

[ ZW/XY ]

[ YX/WZ ]

[ WZ/XY ]

등

겉보기에는 서로 다른 변수 배치가 생성된다.

그러나 이러한 배치들은 모두 동일한 카르노맵 구조를 나타내므로 하나의 동치류로 취급할 수 있다.

이와 같은 관점은 군론에서의 변환군(transform group)과 유사하게 해석할 수 있다. 즉 변수 배치들의 집합 위에 동치 변환들이 작용하며, 서로 변환 가능한 배치들은 동일한 동치류를 형성한다.

다만 본 연구의 목적은 군론적 성질 자체를 분석하는 것이 아니라 카르노맵의 구조적 분류를 수행하는 데 있으므로, 이후에는 이러한 변환들을 동치 변환으로만 다루도록 한다.

X.X.5 변수 배치의 동치류 분류 

앞 절에서 정의한 동치 변환을 이용하여 4변수 카르노맵의 변수 배치를 분류해 보자.

4개의 변수 A, B, C, D를 카르노맵에 배치하는 경우를 생각하면, 변수의 순서까지 모두 고려한 전체 배치의 수는

[ 4! = 24 ]

가지이다.

그러나 앞 절에서 보인 바와 같이 변수의 순서 교환은 카르노맵의 구조를 변화시키지 않는다.

즉

[ XY/ZW \equiv YX/ZW ]

[ XY/ZW \equiv XY/WZ ]

가 성립하므로, 변수의 순서는 더 이상 중요한 요소가 아니다.

따라서 변수 배치를 분석할 때는 어떤 변수가 행에 배치되고 어떤 변수가 열에 배치되는지만 고려하면 충분하다.

이 경우 문제는 4개의 변수 중 행에 배치할 두 변수를 선택하는 문제로 바뀐다.

따라서 가능한 변수 분할은

[ \binom{4}{2}=6 ]

가지이며, 다음과 같이 나타낼 수 있다.

[ AB/CD ]

[ AC/BD ]

[ AD/BC ]

[ BC/AD ]

[ BD/AC ]

[ CD/AB ]

[6개의 변수 분할 시각자료]

그러나 앞 절에서 정의한 축 교환 변환에 의해

[ XY/ZW \equiv ZW/XY ]

가 성립한다.

따라서 위 여섯 개의 변수 분할은 다음과 같이 서로 대응된다.

[ AB/CD \equiv CD/AB ]

[ AC/BD \equiv BD/AC ]

[ AD/BC \equiv BC/AD ]

[축 교환에 의한 대응 관계 시각자료]

결과적으로 여섯 개의 변수 분할은 세 개의 동치류로 축약된다.

즉 4변수 카르노맵에서 고려해야 할 대표 변수 배치는 다음 세 가지뿐이다.

[ AB/CD ]

[ AC/BD ]

[ AD/BC ]

[6개의 변수 분할이 3개의 동치류로 축약되는 시각자료]

이는 변수의 이름과 순서에 관계없이 4변수 카르노맵의 구조가 본질적으로 세 가지 형태로 분류될 수 있음을 의미한다.

따라서 이후의 패턴 분석, XOR/XNOR 구조 분석 및 대칭성 연구에서는 위 세 가지 대표 배치만을 고려하여도 모든 경우를 포괄할 수 있다.

본 연구에서는 이 세 가지 대표 배치를 기준으로 카르노맵의 시각적 패턴과 구조적 특성을 분석한다.
따라서 이후의 분석에서는
AB/CD, AC/BD, AD/BC의 세 가지 대표 배치만을 고려한다.






X.X.6 소결론 

본 절에서는 4변수 카르노맵의 변수 배치를 구조적 관점에서 분석하고, 변수 배치의 동치류 개념을 정의하였다.

먼저 카르노맵의 좌표계와 그레이코드 배열 순서를 정의하고, 카르노맵의 상하 및 좌우 경계가 연결되어 있다는 성질을 바탕으로 순환 이동을 구조 보존 변환으로 정의하였다.

이후 행 변수 순서 교환, 열 변수 순서 교환 및 축 교환을 각각 분석하였다. 그 결과

[ XY/ZW \equiv YX/ZW ]

[ XY/ZW \equiv XY/WZ ]

[ XY/ZW \equiv ZW/XY ]

가 성립함을 보였으며, 이를 카르노맵의 동치 변환으로 정의하였다.

이러한 동치 변환을 이용하면 변수 순서를 포함하여 총 24가지 존재하는 4변수 카르노맵의 배치는 구조적으로 중복되는 경우를 제거할 수 있다. 또한 변수의 순서를 무시하고 변수 집합의 분할만 고려하면 가능한 배치는 여섯 가지로 정리되며, 축 교환에 의한 동치 관계를 적용하면 최종적으로 세 개의 대표 배치로 분류된다.

즉 4변수 카르노맵의 변수 배치는 형식적으로는 다양한 형태를 가질 수 있으나, 구조적 관점에서는 다음 세 가지 대표 배치만을 고려하면 충분하다.

[ AB/CD ]

[ AC/BD ]

따라서 이후의 카르노맵 분석에서는 위 세 가지 대표 배치를 기준으로 논의를 전개하며, 동치 변환에 의해 얻어지는 다른 배치들은 동일한 구조를 나타내는 경우로 간주한다.

본 절에서 제시한 동치류 개념은 변수 배치에 따른 시각적 구조를 체계적으로 분류하기 위한 기준을 제공하며, 이후의 패턴 분석 과정에서 반복되는 구조를 효율적으로 정리할 수 있는 이론적 기반이 된다.
